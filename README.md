# Крестики-нолики (Tic-Tac-Toe)

## 1. Описание игры
- **Правила игры:** Классическая игра для двух игроков на поле 3×3. Игроки поочередно ставят свои символы (`X` или `O`) в пустые клетки.
- **Логика победы/поражения:** Побеждает игрок, первым собравший три своих символа в непрерывный ряд по горизонтали, вертикали или диагонали. Если все клетки заполнены, а победителя нет — фиксируется ничья.
- **Особенности реализации:** Приложение написано на чистом JavaScript без использования фреймворков. Для сетки поля используется CSS Grid, для центрирования символов — Flexbox. Состояние игры отделено от DOM-дерева и хранится в JS-переменных.

## 2. Игровая механика
- **Как определяется победитель?** После каждого хода запускается функция `checkResult()`, которая сравнивает текущее состояние поля с заранее заданным массивом выигрышных комбинаций.
- **Как обрабатываются клики?** На каждую ячейку (`.cell`) навешан обработчик `click`. При клике проверяется активность игры (`gameActive`) и пустота клетки. Если ход допустим, обновляется массив состояния и визуальное отображение, после чего вызывается проверка результата.
- **Как работает перезапуск?** Кнопка "Начать заново" вызывает функцию `restartGame()`, которая сбрасывает все переменные в начальное состояние, очищает текст из ячеек и обновляет статус-текст.

## 3. Структура данных
- **Массив `board`:** Одномерный массив из 9 строк. Индексы `0–8` соответствуют ячейкам поля (слева направо, сверху вниз). Хранит `'X'`, `'O'` или `''` (пусто).
- **Условия победы:** Массив `winningConditions` содержит 8 вложенных массивов с индексами выигрышных линий: 3 горизонтали, 3 вертикали, 2 диагонали.
- **Состояние игры:** Хранится в трёх переменных:
  - `currentPlayer` — текущий ход (`"X"` или `"O"`).
  - `board` — логическое представление поля.
  - `gameActive` — булев флаг, блокирующий ходы после победы/ничьей.

## 4. Алгоритмы
- **Алгоритм проверки победы:** Цикл `for...of` проходит по всем комбинациям из `winningConditions`. Если `board[a] === board[b] && board[b] === board[c] && board[a] !== ''`, устанавливается флаг `roundWon = true`.
- **Алгоритм определения ничьей:** Если `roundWon === false` и метод `board.every(cell => cell !== '')` возвращает `true`, значит все клетки заняты и победа не достигнута → ничья.
- **Алгоритм смены игрока:** Тернарный оператор `currentPlayer === "X" ? "O" : "X"` меняет значение переменной и обновляет UI-статус.
